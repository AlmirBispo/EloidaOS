🔧 Guia de Implementação

1. Pré-requisitos

1
Hardware: Placa CH32V307VCT6 (ou qualquer RISC-V compatível). Também funciona em STM32, ESP32-C3, etc.
2
Toolchain: riscv64-unknown-elf-gcc ou MounRiver Studio (para CH32V).
3
Debugger: WCH-LinkE (RISC-V) ou J-Link com suporte RISC-V.
4
Editor: VSCode + extensão Embedded IDE, ou Eclipse CDT.

2. Estrutura mínima do projeto

projeto/
├── src/
│   ├── main.c                  (seu aplicativo)
│   ├── eloida_kernel.c         (copie o kernel aqui)
│   └── eloida_kernel.h         (header opcional)
├── startup/
│   └── startup_ch32v307.S      (arquivo de startup do MCU)
├── linker/
│   └── ch32v307.ld             (script de linker)
└── Makefile

3. Primeiro exemplo funcional

3.1 Copie o kernel para eloida_kernel.c

Cole o conteúdo integral de ELOIDA-OS-COMPARTILHAMENTO-MEMORIA.c.

3.2 Crie sua própria tarefa

// minha_task.c
#include "eloida_kernel.h"

// Exemplo: pisca LED no pino PA5
extern SharedData SHARED;  // declarado no kernel

void task4(void) {
    // Sua lógica customizada
    GPIOA->ODR ^= GPIO_PIN_5;   // toggle LED
    SHARED.debug_log[0] = 'T';
    SHARED.debug_log[1] = '4';
}

3.3 Inicialize o hardware antes do main loop

void SystemClock_Config(void) {
    // Configure o clock para 144 MHz (CH32V307)
    // ...
}

void GPIO_Init(void) {
    // Habilita clock do GPIOA
    RCC->APB2PCENR |= RCC_APB2Periph_GPIOA;
    // Configura PA5 como saída push-pull
    GPIOA->CFGLR &= ~(0xF << (5*4));
    GPIOA->CFGLR |=  (0x3 << (5*4));
}

int main(void) {
    SystemClock_Config();
    GPIO_Init();
    
    // Loop principal do ELOIDA OS (já vem no kernel)
    while(1) {
        for (u8 i = 0; i < MAX_PROCESSES; i++) {
            process((i < 3) ? PRIME_NUMS[i] : (1000 + i), i);
            run(i);
            destroy(i);
        }
    }
}

4. Compilando com Makefile

CC      = riscv64-unknown-elf-gcc
CFLAGS  = -march=rv32imac -mabi=ilp32 -O2 -Wall
LDFLAGS = -T linker/ch32v307.ld -nostdlib -Wl,--gc-sections

SRCS = src/main.c src/eloida_kernel.c src/minhas_tasks.c \
       startup/startup_ch32v307.S

OBJS = $(SRCS:.c=.o)
OBJS := $(OBJS:.S=.o)

all: eloida.elf eloida.bin

eloida.elf: $(OBJS)
	$(CC) $(CFLAGS) $(LDFLAGS) $^ -o $@

eloida.bin: eloida.elf
	riscv64-unknown-elf-objcopy -O binary $< $@

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

flash: eloida.bin
	wch-openocd -f wch-riscv.cfg -c "program eloida.bin verify reset exit"

clean:
	rm -f src/*.o startup/*.o eloida.elf eloida.bin

5. Gravando no MCU

Opção A: WCH-LinkE (recomendado para CH32V)

# Via MounRiver Studio (Windows/Linux)
# 1. Importar projeto
# 2. Build → Clean + Build
# 3. Debug → Launch Mode

# Via wch-openocd (linha de comando)
wch-openocd -f wch-riscv.cfg \
  -c "program build/eloida.bin 0x00000000 verify reset exit"

Opção B: DFU (bootloader USB do CH32V307)

# Colocar o MCU em modo DFU (pressionar BOOT0 + RESET)
wchisp flash eloida.bin

6. Depurando tarefas

✅ Use o campo debug_log[16] da SharedData
Cada tarefa pode escrever mensagens curtas que serão enviadas via UART por uma task dedicada. 
// task10.c — Logger UART
void task10(void) {
    static u8 last_idx = 0;
    if (SHARED.debug_log[0] != 0) {
        UART_Send(SHARED.debug_log, 16);
        memset(SHARED.debug_log, 0, 16);
    }
}
⚠️ Erros comuns:
  • Esquecer de habilitar o clock do periférico (RCC)
  • Tarefas que bloqueiam (ex: delay_ms) — use polling
  • Modificar SHARED fora de uma task (quebra o modelo cooperativo)
  • Usar printf sem UART configurado trava o sistema
