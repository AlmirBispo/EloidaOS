🌿 ELOIDA OS

Sistema Operacional baseado em números primos para economia extrema de recursos

Autor: Almir Bispo · RISC-V · CH32V307VCT6 · 2026

🧮 Fundamentos 🏗️ Arquitetura 🔧 Implementação 📈 Expansão ⭐ GitHub

🎯 Propósito

O ELOIDA OS nasceu da necessidade de executar múltiplas tarefas concorrentes em microcontroladores com poucos kilobytes de RAM, sem recorrer a RTOS tradicionais que exigem pilhas separadas, troca de contexto e alocação dinâmica.

A sacada: usar números primos como intervalos de despacho. Como dois primos distintos são sempre coprimes (MDC = 1), duas tarefas jamais entram em "ressonância" — eliminando o fenômeno de colisão periódica que aflige schedulers baseados em potências de 2.

✨ Características

⚡ Zero Overhead

Tabela de despacho estática resolvida em tempo de compilação. Sem ponteiros de pilha, sem salvamento de contexto.

🔒 IPC Seguro

Memória compartilhada tipada (SharedData). Como o scheduler é cooperativo e determinístico, mutexes são desnecessários.

📐 255 Processos

Suporta até 255 tarefas simultâneas com apenas 255 bytes de flags + 1 KB de contadores.

🧠 Matemática Pura

Os primeiros 255 números primos pré-computados garantem distribuição ótima de CPU.

🎛️ Bare-Metal

Roda direto no CH32V307VCT6 (RISC-V) sem dependências externas. Portável para qualquer MCU.

📦 12 Tarefas Nativas

Tabela inicial com 12 slots: sensor → filtro → atuador → status + 8 customizáveis.
